The Human Design API is designed for programmatic consumption: structured JSON, a comprehensive OpenAPI spec, and deterministic results. This makes it straightforward to integrate as a tool for AI agents and LLM workflows.

## Why This API Works Well with AI

- **Structured JSON** — consistent response format with `snake_case` fields
- **OpenAPI spec** — import directly into tool-building frameworks
- **Deterministic** — same birth data always produces the same chart

## OpenAI Function Calling

Define the API as a tool in the OpenAI function calling format:

```json
{
  "type": "function",
  "function": {
    "name": "generate_human_design_chart",
    "description": "Generate a Human Design personality chart from birth data. Returns type, profile, gates, channels, and centers.",
    "parameters": {
      "type": "object",
      "properties": {
        "birthdate": {
          "type": "string",
          "description": "Date of birth in DD-MMM-YY format (e.g., 15-Jan-90)"
        },
        "birthtime": {
          "type": "string",
          "description": "Time of birth in HH:MM 24-hour format"
        },
        "location": {
          "type": "string",
          "description": "Birth location as city and country"
        }
      },
      "required": ["birthdate", "birthtime", "location"],
      "additionalProperties": false
    },
    "strict": true
  }
}
```

When the LLM invokes this tool, make the API call:

<CodeGroup>
```javascript JavaScript
const response = await fetch("https://api.humandesignapi.nl/v1/bodygraphs/simple", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "HD-Api-Key": "YOUR_API_KEY",
    "HD-Geocode-Key": "YOUR_GEOCODE_KEY",
  },
  body: JSON.stringify({ birthdate, birthtime, location }),
});

const result = await response.json();
return result;
```

```python Python
import requests

response = requests.post(
    "https://api.humandesignapi.nl/v1/bodygraphs/simple",
    headers={
        "Content-Type": "application/json",
        "HD-Api-Key": "YOUR_API_KEY",
        "HD-Geocode-Key": "YOUR_GEOCODE_KEY",
    },
    json={"birthdate": birthdate, "birthtime": birthtime, "location": location},
)

result = response.json()
return result
```
</CodeGroup>

## No-Code Platforms (Zapier, Make, n8n)

Use an HTTP request module with these settings:

| Setting | Value |
| --- | --- |
| **Method** | POST |
| **URL** | `https://api.humandesignapi.nl/v1/bodygraphs/simple` |
| **Headers** | `HD-Api-Key: YOUR_API_KEY`, `HD-Geocode-Key: YOUR_GEOCODE_KEY`, `Content-Type: application/json` |
| **Body** | `{"birthdate": "DD-MMM-YY", "birthtime": "HH:MM", "location": "City, Country"}` |

Map dynamic fields from your workflow into the body. The response contains the chart data directly as a JSON object.

## Recommended Endpoints for Agents

| Use Case | Endpoint | Why |
| --- | --- | --- |
| Quick personality summary | `POST /v1/bodygraphs/simple` | Smaller response, all plan tiers |
| Full chart reading | `POST /v1/bodygraphs` | All properties, advanced tier |

## Error Handling for Agents

Always check the HTTP status code and handle errors gracefully:

<CodeGroup>
```javascript JavaScript
if (!response.ok) {
  const error = await response.json();
  if (response.status === 402) {
    return "Credit limit reached. Please try again later.";
  } else if (response.status === 400) {
    return "Invalid birth data. Please check the format.";
  } else {
    return `API error: ${response.status}`;
  }
}
```

```python Python
if not response.ok:
    if response.status_code == 402:
        return "Credit limit reached. Please try again later."
    elif response.status_code == 400:
        return "Invalid birth data. Please check the format."
    else:
        return f"API error: {response.status_code}"
```
</CodeGroup>
